1부에서 국민건강영양조사 데이터로 뇌졸중 예측 모델을 만들었다. AUC는 0.78~0.82로 다섯 개 모델 모두 준수했는데, 정작 threshold 0.5에서 예측을 시켜보면 로지스틱·랜덤포레스트·그래디언트부스팅은 뇌졸중 사례를 단 한 건도 양성으로 못 잡아낸다. recall이 0이다. 이 글에서는 이 문제를 어떻게 다뤘는지, 그리고 "recall을 우선시하는 스크리닝 모델링"이 정말 정답인지를 다룬다.

## 왜 threshold 0.5에서 recall이 0이 되는가

로지스틱 회귀를 예로 들면, threshold=0.5에서의 혼동행렬은 이렇다.

```
                 Predicted 0   Predicted 1
Actual 0 (7607)       7607          0
Actual 1 (227)         227          0

Recall = 0.00 / Precision = 0.00
```

뇌졸중 사례 227건이 전부 음성으로 분류됐다. 모델이 멍청해서가 아니다. 학습 데이터에서 뇌졸중 비율이 2.89%밖에 안 되니, 로지스틱 회귀의 결정 경계 입장에서는 "무조건 정상이라고 찍는" 전략만으로도 전체 정확도 97%를 손쉽게 달성할 수 있다. threshold=0.5는 "두 클래스가 반반일 때" 의미가 있는 기준점이지, 유병률 2.89%인 데이터에는 애초에 맞지 않는 기준이다.

1차 스크리닝이 목적이라면 이 상태의 모델은 그대로 쓸 수 없다. 위음성(뇌졸중인데 놓치는 경우)의 비용이 위양성(정상인데 재검사를 권하는 경우)보다 훨씬 크기 때문에, threshold를 낮춰서 recall을 끌어올리는 작업이 필요하다.

## Threshold를 낮추면 실제로 무슨 일이 벌어지는가

Precision-Recall 곡선으로 전체 궤적을 먼저 봤다.

![Precision-Recall 곡선 비교, 뇌졸중 클래스 기준 5개 모델](/assets/images/posts/stroke-prediction-2-pr-curve.jpg)

모든 모델이 무작위 기준선(유병률 2.89%)보다는 위에 있지만, recall을 올릴수록 precision이 가파르게 떨어지는 모양이 공통적으로 나타난다. 로지스틱 회귀 하나를 잡고 threshold를 0부터 끝까지 움직이면서 precision과 recall이 어떻게 바뀌는지 그려봤다.

```python
import numpy as np
from sklearn.metrics import precision_recall_curve

precisions, recalls, thresholds = precision_recall_curve(y_true=y_test, probas_pred=y_proba)

# recall 0.8 이상인 지점 중 precision이 가장 높은 지점을 채택
valid_indices = np.where(recalls[:-1] >= 0.8)[0]
best_idx = valid_indices[-1]
best_threshold = thresholds[best_idx]
```

![threshold 이동에 따른 precision-recall 트레이드오프, 로지스틱 회귀 기준](/assets/images/posts/stroke-prediction-2-threshold-tradeoff.jpg)

recall을 0.8까지 끌어올리려면 threshold를 0.028까지 낮춰야 한다. 기본값 0.5의 18분의 1 수준이다. 이 지점에서 precision은 0.072다. 모델이 "뇌졸중 위험"이라고 지목한 사람 100명 중 실제 뇌졸중 진단자는 7명뿐이라는 뜻이다.

## Recall 0.8, Precision 0.07의 혼동행렬

같은 로지스틱 모델을 threshold 0.5와 0.028 두 지점에서 나란히 비교하면 이렇다.

![threshold 조정 전후 혼동행렬 비교, 기본 threshold 0.5와 recall 타겟팅 threshold 0.028](/assets/images/posts/stroke-prediction-2-confusion-before-after.jpg)

threshold를 0.028로 낮추면 실제 뇌졸중 227명 중 182명을 잡아낸다(recall 0.80). 대신 정상군 7607명 중 2339명이 양성으로 분류된다. 뇌졸중이 아닌데 재검사 대상으로 지목되는 사람이 실제 뇌졸중 환자보다 12배 이상 많아지는 셈이다.

다섯 개 모델 전체에 같은 방식(recall 0.8 지점 중 precision이 가장 높은 threshold)을 적용해보면 결과가 거의 똑같이 수렴한다.

```
model             threshold  recall  precision  f1
Logistic             0.0276   0.802      0.072  0.132
RandomForest          0.0345   0.802      0.073  0.133
XGBoost               0.0295   0.802      0.073  0.135
GradientBoosting      0.0286   0.802      0.072  0.132
NaiveBayes            0.0253   0.802      0.066  0.122
```

모델 종류와 무관하게 recall 0.8 근처에서 precision이 0.07 안팎으로 수렴한다. 이건 모델 성능의 문제라기보다, 애초에 유병률 2.89%인 문제에서 recall 80%를 요구하면 precision이 이 언저리로 떨어질 수밖에 없다는 걸 보여준다. 모델을 바꿔도 이 트레이드오프 자체를 벗어나긴 어렵다.

## Recall 우선 전략은 정형화된 패턴인가

여기서 처음 던졌던 질문으로 돌아가게 된다. "불균형 데이터는 recall을 최우선으로 해서 1차 스크리닝 목적의 모델링을 하는 게 현실적으로 맞는가"라는 질문이었는데, 방향 자체는 맞다고 본다. 위음성의 비용이 위양성보다 훨씬 큰 도메인에서는 threshold를 낮춰 recall을 확보하는 게 타당하다.

다만 "threshold를 낮춰서 recall을 올린다"는 조작 자체는 위에서 본 것처럼 기계적으로 반복 가능한 절차다. 어느 모델을 쓰든 recall 0.8 지점의 threshold를 코드 몇 줄로 찾아낼 수 있다. 진짜 판단이 필요한 지점은 그 다음이다.

첫째, recall을 어디까지 올릴지는 임의로 정할 수 없다. 이번 예시에서 쓴 recall 0.8이라는 목표치도 사실 자의적으로 잡은 값이다. 실제로는 위음성 비용과 위양성 비용의 비율, 혹은 2차 검진으로 넘길 수 있는 실제 리소스(capacity)에서 threshold가 역산되어야 한다. precision 0.07이라는 숫자가 감당 가능한지 여부는 뇌졸중 데이터 자체가 아니라 다운스트림 의료 체계의 용량에 달려 있다.

둘째, recall 하나만 보고 threshold를 정당화할 수 없다. threshold를 0에 가깝게 내리면 recall은 기계적으로 1에 수렴한다. 이번 실험에서도 threshold를 더 낮추면 recall은 계속 올라가지만 precision은 0.03, 0.02로 더 떨어진다. recall이 목표치에 도달했다고 끝나는 게 아니라, 그 지점의 precision이 "feasible"한지를 함께 봐야 완결된 판단이 된다.

셋째, 이 글에서는 다루지 않았지만 연령대나 성별 같은 하위 집단별로 같은 threshold에서 recall이 고르게 나오는지도 확인이 필요하다. 전체 recall이 0.8이어도 특정 연령대에서만 낮게 나온다면 그 threshold는 절반만 맞는 답이다.

정리하면, threshold 이동을 통한 recall 확보는 기법 자체는 정형화되어 있지만, 그 threshold를 어디서 멈출지를 결정하는 근거(비용비, capacity, 하위집단 검증)는 도메인마다 달라지는 비정형적 작업이다. "recall이 답이다"보다는 "recall 수치 하나만으로는 threshold를 정당화할 수 없다"는 쪽이 더 정확한 결론이라고 본다.

## 현장에 적용한다면: Threshold가 실제로 의미하는 것

recall 0.8, precision 0.07이라는 숫자를 실제 현장의 언어로 바꿔보면 이렇다. 테스트셋 7,834명 중 실제 뇌졸중 진단자는 227명이었고, threshold 0.028을 적용하니 182명을 잡아내고 45명을 놓쳤다. 대신 정상인 7,607명 중 2,339명이 양성으로 분류됐다. 비율로 정리하면, 뇌졸중 환자 1명을 추가로 찾아낼 때마다(182명 기준) 정상인 12.8명(2,339 ÷ 182)이 불필요하게 재검사 대상으로 지목된다는 뜻이다.

이 숫자가 현장에서 감당 가능한지는 이 프로젝트의 데이터만으로는 판단할 수 없다. 판단하려면 최소한 세 가지 정보가 더 필요하다. 첫째, 2차 검진(예: 뇌 MRI, 경동맥 초음파 등)을 추가로 얼마나 소화할 수 있는지에 대한 실제 검진 인프라의 여유 capacity. 둘째, 위양성 1건이 발생시키는 비용과 부담(검사 비용, 대기 시간, 불필요한 불안감 같은 심리적 비용까지 포함). 셋째, 반대로 위음성 1건, 즉 뇌졸중을 놓쳤을 때 발생하는 사회적·의료적 비용이다. 국민건강영양조사는 설문 기반 단면조사이기 때문에 이 세 가지 중 어느 것도 담고 있지 않다.

즉 "recall 0.8을 목표로 잡는다"는 결정 자체가 이미 암묵적으로 어떤 capacity와 비용 구조를 가정하고 있는 셈인데, 그 가정이 실제로 맞는지는 이 데이터셋 밖에서 검증해야 한다. 정상인 12.8명당 환자 1명을 더 찾아내는 트레이드오프가 어떤 현장에서는 당연히 받아들일 만한 비용이고, 다른 현장에서는 검진 시스템을 마비시키는 부담일 수 있다. 모델링 결과만 놓고 "이 threshold가 현장에 쓸 수 있다/없다"를 판단하는 건 이 시점에서는 성급하다. 이 프로젝트가 낼 수 있는 정직한 결론은 "이 트레이드오프가 얼마나 비싼지를 숫자로 보여줬다"까지고, "그 비용을 감당할 수 있는가"는 검진 현장의 운영 데이터를 가진 쪽이 답해야 할 질문이다.

## 정리

로지스틱 회귀 기준으로 threshold를 0.5에서 0.028까지 낮추면 recall은 0에서 0.80으로 오르지만 precision은 0.07까지 떨어진다. 다섯 개 모델 모두 recall 0.8 지점에서 precision이 0.07 안팎으로 수렴했다. threshold 이동 자체는 모델과 무관하게 반복 가능한 패턴이지만, 어디까지 recall을 밀어붙일지는 비용함수와 운영 제약에서 나와야 한다.

그런데 다른 방법도 있다. threshold를 건드리는 대신, 애초에 학습 데이터의 클래스 비율 자체를 SMOTE나 언더샘플링으로 맞추면 recall이 훨씬 쉽게 오르지 않을까. 3부에서는 이 방법을 실제로 테스트하고, 그렇게 얻은 recall 상승이 threshold 이동과 근본적으로 같은 것인지 다른 것인지를 다룬다.
